# Release Spotlight Format

## Goal

Turn a month of merged work into a readable product update.

## Good Structure

Use this shape by default:

1. Short title
2. One-line description
3. Intro with exact date range
4. `## What Shipped`
5. 4-6 sections grouped by theme
6. Model-release CTA block
7. `## Shipping Snapshot`
8. `## Next Up`

## Section Patterns

Use headings like:

- `Video, async jobs, and observability rolled out`
- `Provider activation and pricing got clearer`
- `TypeScript Agent SDK launched`
- `Model coverage kept moving`
- `Late-month cleanup and reliability refresh`

Avoid headings like:

- `PR Updates`
- `Misc fixes`
- `Internal improvements`
- `Data and product changes`

## CTA Rules

For model discovery links, prefer:

- `Explore models here`
- `Browse the latest releases`
- `Explore recent updates here`

Avoid visible link text like:

- `/models`
- `/updates`

## Model Section Guidance

- Mention the biggest releases by name.
- Do not try to enumerate every model unless that is the point of the post.
- Keep withheld, rumoured, or inferred-date nuance in the changelog when possible.

## Tone

- Direct
- Product-facing
- Concrete
- Slightly editorial, but not hype-heavy

## Final Check

Ask:

- Does the post have a clear narrative?
- Are the biggest launches easy to spot?
- Are model links human-readable?
- Would this still make sense if read without the PR list?
