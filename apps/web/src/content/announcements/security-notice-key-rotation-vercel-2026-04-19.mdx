We are issuing this notice in response to Vercel's security bulletin published on **April 19, 2026**:

- [Vercel April 2026 security incident](https://vercel.com/kb/bulletin/vercel-april-2026-security-incident)

## What this means for AI Stats users

Out of caution, we are recommending that **all users rotate their keys as soon as possible**.

Even if you have not seen any suspicious activity, this is a precautionary security step we recommend taking now.

## What we changed

As a precaution, we have **updated our key creation flow** to strengthen key safety during this period.

## What you should do now

1. Create a new key in [/settings/keys](/settings/keys).
2. Update your apps, services, and CI/CD secrets to use the new key.
3. Verify traffic is working on the new key.
4. Revoke old keys immediately after cutover.
5. Review usage/activity logs for unexpected requests.

<Callout title="Priority" tone="warning">
Rotate production keys first, then staging and development keys.
</Callout>

## Recommended rotation checklist

- API keys used in backend services
- Keys used in CI/CD pipelines
- Keys stored in environment variables
- Shared team keys or long-lived service keys

## We will continue to monitor

We are actively monitoring and will publish follow-up guidance if anything changes.
