Phaseo now has a clearer separation between credentials that send inference
requests and credentials that manage a workspace.

This release also adds a Raycast extension for the information developers tend
to check most often: model and provider discovery, current credits, and recent
gateway activity.

## Management access now has its own key type

Gateway API keys remain `phaseo_v1_sk_...` credentials. They are for inference
requests and continue to use the key-level policies and guardrails attached to
them.

Management API keys now use `phaseo_v1_mk_...`. They are for control-plane
operations such as viewing credits and activity, managing workspace settings,
or automating key and preset administration. A management key cannot be used
for gateway inference, and an inference key cannot be used for management
endpoints.

That split removes an easy-to-make configuration mistake: pasting a regular
gateway key into an administration integration and expecting it to have
elevated access.

## Start with the access level, not a blank permissions list

When creating a management key in Settings, choose an access level that fits
the integration:

- **Raycast** can read credits and recent activity only.
- **Read** can inspect the current management surfaces.
- **Write** adds updates, without delete access.
- **All** includes the current read, write, and delete capabilities for trusted
  administrative automation.

Each option expands to an explicit set of capabilities when the key is
created. New capabilities are never added to an existing key implicitly.

## OAuth apps keep approvals explicit too

OAuth app owners can set the capabilities an app may request. Each user and
workspace keeps its own approved grant, so changing an app's allowed scopes
does not silently expand an existing authorization.

If an app starts requesting additional access, the dashboard shows that there
are unapproved scopes. The user can review and approve them in the normal OAuth
flow, or leave the existing grant unchanged.

## Raycast gets the useful, low-risk views

The Phaseo Raycast extension now includes:

- model, organisation, and provider browsing
- Usage & Credits
- Recent Gateway Activity
- cached catalogue and account reads to avoid unnecessary API traffic

The catalogue commands use the regular gateway API key configured in Raycast.
Usage and activity use a separate Raycast-scoped management key, so the
extension gets only the two read capabilities it needs.

If a regular `sk` key is entered in the management-key preference, Raycast
explains that an `mk` management key is required before making an API request.

## Safer automation without a slower workflow

The intent is not to make automation harder. It is to make its boundary clear:
use a narrow management key for the control-plane task, a regular gateway key
for inference, and OAuth when an application needs a user-delegated grant.

That gives teams more confidence to add useful surfaces such as Raycast without
turning a small read-only integration into an all-access credential.
