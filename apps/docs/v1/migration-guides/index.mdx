# Migration Guides

Welcome to the AI Stats migration guides. These guides provide step-by-step instructions to help you migrate your integrations, whether upgrading between versions of the AI Stats API and SDKs or migrating from other AI platforms and gateways.

## Types of Migration Guides

We provide two main categories of migration guides:

### 1. AI Stats Breaking Changes

These guides help you upgrade between different versions of the AI Stats platform when we introduce breaking changes. They cover:

-   **Breaking changes**: Updates that may require code modifications
-   **New features**: How to take advantage of new capabilities
-   **Deprecated functionality**: What to replace and how
-   **Best practices**: Recommendations for smooth upgrades

Each guide includes:

-   Overview of changes
-   Detailed migration steps
-   Code examples
-   Troubleshooting tips

### 2. Migrating from Other Platforms

These guides help you migrate your existing integrations from other AI platforms, SDKs, and gateways to AI Stats. We currently provide guides for:

-   [OpenAI SDK Migration](from-openai)
-   [Anthropic SDK Migration](from-anthropic)
-   [OpenRouter Migration](from-openrouter)
-   [Vercel AI Gateway Migration](from-vercel)
-   [LiteLLM Migration](from-litellm)
-   [Cloudflare AI Gateway Migration](from-cloudflare)
-   [Helicone AI Gateway Migration](from-helicone)

## How to Use These Guides

1. **Identify your migration type**: Determine if you're upgrading AI Stats versions or migrating from another platform.
2. **Find the relevant guide**: Select the migration guide that matches your scenario.
3. **Follow the steps sequentially**: Execute the changes in the order provided.
4. **Test thoroughly**: Verify your integration works after migration.
5. **Refer to the API reference**: Use the updated API documentation for implementation details.
