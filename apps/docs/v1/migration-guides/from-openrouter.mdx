title: "Migration from OpenRouter"

---

If your app currently uses a gateway like OpenRouter, this guide shows the shortest migration path to AI Stats Gateway.

## What changes

1. Swap base URL.
2. Swap API key.
3. Confirm model IDs.
4. Remove any gateway-specific headers you no longer need.

For most OpenAI-compatible clients, request and response shapes remain the same.

## 1) Base URL and key swap

<CodeGroup>
```bash cURL
# Before
curl https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer sk-or-v1-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello"}]
  }'

# After
curl https://api.phaseo.app/v1/chat/completions \
  -H "Authorization: Bearer aistats_v1_sk_<kid>_<secret>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

```python Python
from openai import OpenAI

# Before
before = OpenAI(
    api_key="sk-or-v1-...",
    base_url="https://openrouter.ai/api/v1",
)

# After
after = OpenAI(
    api_key="aistats_v1_sk_<kid>_<secret>",
    base_url="https://api.phaseo.app/v1",
)
```

```typescript TypeScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.AI_STATS_API_KEY,
  baseURL: "https://api.phaseo.app/v1",
});
```
</CodeGroup>

## 2) Model IDs

Most provider-prefixed IDs continue to work (for example `openai/gpt-4o-mini`, `anthropic/claude-sonnet-4.5`).

Always verify available IDs via:

- [Models endpoint](../api-reference/endpoint/models.mdx)
- [Models explorer](https://ai-stats.phaseo.app/models)

## 3) Free model behavior

AI Stats supports free-model IDs that end in `:free`.

- `:free` models can be called with zero deposited credits.
- Paid models require available wallet balance.

## 4) Common migration checks

- Ensure all requests send `Authorization: Bearer <AI_STATS_API_KEY>`.
- Remove legacy gateway-specific headers unless you explicitly need them.
- Test one text route first (`/v1/responses` or `/v1/chat/completions`) before migrating image/audio/video routes.
- Validate tool-calling and structured-output behavior in staging with your real prompts.

## 5) Suggested rollout

1. Route a small percentage of traffic to AI Stats.
2. Compare latency, error rates, and output quality.
3. Increase traffic gradually.
4. Keep a rollback toggle until parity is confirmed.
