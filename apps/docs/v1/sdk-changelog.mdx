<Update label="1st May 2026">
# SDKs 2.0.3 Release

This release tracks the `2.0.3` SDK line across TypeScript, Python, Go, C#, Java, PHP, and Ruby.

This release includes:

- Auto-release wiring for functional SDK packages after OpenAPI or model-surface changes, so published clients stay closer to current gateway behavior.
- Regenerated types and model enums for the latest API and catalog updates.
- Continued support for the same functional SDK set, while C++ and Rust remain outside this auto-release line for now.

</Update>

<Update label="29th April 2026">
# SDKs 2.0.2 Release

This release tracks the `2.0.2` SDK line across TypeScript, Python, Go, C#, Java, PHP, and Ruby.

This release includes:

- Request-side model identifiers are now treated as runtime strings, so newly released models can be used without waiting for a fresh SDK release.
- Generated helper constants now follow the callable-on-gateway snapshot instead of the full catalog.
- Model-helper churn is treated as patch-level data change instead of forcing larger version jumps.

</Update>

<Update label="22nd April 2026">
# SDKs 2.0.0 Release

This release starts the `2.0.0` major SDK line across TypeScript, Python, Go, C#, Java, PHP, and Ruby.

This release includes:

- Regenerated clients aligned with the latest gateway API and current cross-language endpoint behavior.
- Broader parity work across models, pricing, workspaces, migration guides, and video or batch flows.
- Breaking SDK surface updates tied to the current gateway contract and generated model surfaces.

</Update>

<Update label="18th April 2026">
# SDKs 1.2.0 Release

This release tracks the `1.2.0` SDK line across TypeScript, Python, Go, C#, Java, PHP, and Ruby.

This release includes:

- Updated SDK model availability metadata so newly released models are reflected in model discovery and typed surfaces.
- Added latest model availability coverage for recent additions including Gemma 4 (26B A4B, 31B), GLM 5V Turbo, Qwen 3.6 Plus, and other current catalog updates.
- Synced generated SDK types and model references with the latest API/catalog changes to reduce model ID drift.
- Refreshed SDK docs/examples to align with the current model set available through the gateway.

The goal is to keep SDK model availability aligned with production catalog updates and reduce integration friction when new models launch.

</Update>

<Update label="22nd March 2026">
# SDKs 1.1.0 Release

This release tracks the `1.1.0` SDK line across TypeScript, Python, Go, C#, Java, PHP, and Ruby.

This release includes:

- Synced model availability surfaces with OpenAPI so typed model and metadata coverage stays aligned with the API contract.
- Introduced model lifecycle warnings to surface deprecation and retirement context during SDK usage.
- Updated various SDK paths and route mappings to match REST API behavior more closely across languages.
- Refreshed docs/examples and expanded smoke/lifecycle coverage to improve integration reliability.

The goal is to make SDK behavior more predictable, safer for model migrations, and more consistent with the REST API.

</Update>
