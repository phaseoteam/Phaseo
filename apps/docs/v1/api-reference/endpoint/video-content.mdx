---
title: "Video content (Beta)"
badge: "Beta"
openapi: "GET /videos/{video_id}/content"
description: "Download the rendered video content for a completed generation."
---
Returns the raw video binary from the upstream provider when the generation is ready.

Use this endpoint when you want authenticated byte access through the API itself.

- It is only available once the job is complete and output access allows direct bytes.
- For browser-friendly or temporary direct downloads, prefer `POST /videos/{video_id}/download_url`.
- For multi-output jobs, inspect `outputs` on the status response before requesting a signed download URL for a specific output index.
