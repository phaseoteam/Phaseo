---
title: "OpenCode"
description: "Connect OpenCode to the AI Stats Gateway with a custom OpenAI-compatible provider."
---

OpenCode can call AI Stats through a custom OpenAI-compatible provider. Use this setup when you want OpenCode model selection, usage, and routing to go through one AI Stats API key.

## Prerequisites

- An AI Stats API key.
- OpenCode installed locally.

## 1) Set your API key

```bash
export AI_STATS_API_KEY="your_ai_stats_api_key"
```

```powershell
$env:AI_STATS_API_KEY = "your_ai_stats_api_key"
```

## 2) Add an OpenCode provider

Create or update `opencode.json` in your project, or your global OpenCode config:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "ai-stats/openai/gpt-5.5",
  "provider": {
    "ai-stats": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "AI Stats",
      "options": {
        "baseURL": "https://api.phaseo.app/v1",
        "apiKey": "{env:AI_STATS_API_KEY}"
      },
      "models": {
        "openai/gpt-5.5": {
          "name": "GPT-5.5"
        },
        "openai/gpt-5.4-mini": {
          "name": "GPT-5.4 Mini"
        },
        "anthropic/claude-sonnet-4.5": {
          "name": "Claude Sonnet 4.5"
        }
      }
    }
  }
}
```

Add or remove models from the `models` object to match the AI Stats model ids you want in OpenCode.

## 3) Start OpenCode

Restart OpenCode after editing the config, then choose the AI Stats model from `/models`.

## Verification

- Run a small prompt before using file-editing workflows.
- If a model is rejected, confirm the exact id with the [Models endpoint](../../api-reference/endpoint/models.mdx).
- If authentication fails, confirm `AI_STATS_API_KEY` is set in the same shell that starts OpenCode.

## Add docs context

To let OpenCode read AI Stats docs during a session, add the [Docs MCP](./docs-mcp.mdx) server as well.
