title: "Tool Calling"
description: "Use model-driven function calls safely through the Gateway."
---

Tool calling lets models request structured actions (for example, database lookups, weather checks, or internal API calls) instead of guessing answers.

The Gateway supports tool payloads across these text endpoints:

- `/v1/chat/completions` (OpenAI-style `tools` and `tool_calls`)
- `/v1/responses` (Responses-style `function_call` output items)
- `/v1/messages` (Anthropic-style `tool_use` blocks)

## Request

```bash
curl https://api.phaseo.app/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-5-nano",
    "messages": [
      { "role": "user", "content": "What is the weather in London?" }
    ],
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "get_weather",
          "description": "Get current weather by city",
          "parameters": {
            "type": "object",
            "properties": {
              "city": { "type": "string" }
            },
            "required": ["city"]
          }
        }
      }
    ],
    "tool_choice": {
      "type": "function",
      "function": { "name": "get_weather" }
    },
    "stream": false
  }'
```

## Response

```json
{
  "id": "chatcmpl_...",
  "object": "chat.completion",
  "choices": [
    {
      "index": 0,
      "finish_reason": "tool_calls",
      "message": {
        "role": "assistant",
        "content": null,
        "tool_calls": [
          {
            "id": "call_123",
            "type": "function",
            "function": {
              "name": "get_weather",
              "arguments": "{\"city\":\"London\"}"
            }
          }
        ]
      }
    }
  ]
}
```

Run your tool, then send the tool result back in the next request so the assistant can finish the answer.

## Built-in server tools

The gateway currently exposes these built-in server tools:

- `gateway:datetime`
- `gateway:web_search`
- `gateway:web_fetch`
- `gateway:apply_patch`

This tool runs on the gateway side (no client-side executor required). The gateway rewrites it into an upstream tool/function call, executes it, and feeds the tool result back into the model loop.

Supported request shape:

```json
{
  "tools": [
    {
      "type": "gateway:datetime",
      "parameters": {
        "timezone": "Europe/London"
      }
    }
  ]
}
```

Notes:

- `parameters.timezone` is optional and must be a valid IANA timezone.
- `timezone` can also be provided as a top-level shortcut on the tool object.
- The result contains ISO datetime plus the resolved timezone.
- Usage includes `usage.server_tool_use.datetime_requests`.
- Prefer `tool_choice: "auto"` so the model can decide when to call it.

### Web search example

```json
{
  "tools": [
    {
      "type": "gateway:web_search",
      "parameters": {
        "engine": "exa",
        "max_results": 5,
        "max_total_results": 20,
        "search_context_size": "medium",
        "include_highlights": true,
        "allowed_domains": ["example.com"],
        "excluded_domains": ["reddit.com"]
      }
    }
  ]
}
```

Notes:

- The model supplies the search query when it calls the tool.
- Usage includes `usage.server_tool_use.web_search_requests`.
- When fetched result text or highlights are returned, usage also includes `usage.server_tool_use.web_fetch_requests`.

### Web fetch example

```json
{
  "tools": [
    {
      "type": "gateway:web_fetch",
      "parameters": {
        "max_chars": 12000
      }
    }
  ]
}
```

Notes:

- The model supplies the target `url` when it calls the tool.
- Only HTTP(S) URLs and text-like content types are supported.
- HTML content is reduced to bounded plain text before being injected back into the model loop.
- Usage includes `usage.server_tool_use.web_fetch_requests`.

### Apply patch example

```json
{
  "tools": [
    {
      "type": "gateway:apply_patch"
    }
  ]
}
```

Notes:

- The model supplies a `patch` string when it calls the tool.
- The patch must use the Codex-style `*** Begin Patch` / `*** End Patch` envelope.
- AI Stats validates and returns the patch as an artifact, but does not apply files on the server.
- Usage includes `usage.server_tool_use.apply_patch_requests`.

### Image generation example

```json
{
  "tools": [
    {
      "type": "gateway:image_generation",
      "parameters": {
        "model": "openai/gpt-image-latest"
      }
    }
  ]
}
```

The model supplies the image prompt when it calls the tool. AI Stats runs the image request through the normal image generation endpoint, so provider routing, pricing, billing, and audit behavior match a direct `/v1/images/generations` request. Usage includes `usage.server_tool_use.image_generation_requests`.

### Fusion example

```json
{
  "tools": [
    {
      "type": "gateway:fusion",
      "parameters": {
        "analysis_models": ["openai/gpt-5.5"],
        "include_web": true
      }
    }
  ]
}
```

Fusion lets the model request a bounded multi-model analysis panel when the task needs a second opinion or structured critique. Inner analysis calls are normal gateway requests and may use gateway web search/fetch when enabled. Usage includes `usage.server_tool_use.fusion_requests`.

### Tool search example

```json
{
  "tools": [
    {
      "type": "gateway:tool_search"
    }
  ]
}
```

Tool search returns matching AI Stats server-tool definitions from the local gateway catalog. Usage includes `usage.server_tool_use.tool_search_requests`.

## Streaming behavior

Tool-calling requests can also use `stream: true`.

For gateway-managed server tools, the gateway may:

- materialize the upstream tool-call turn
- execute the server tool
- continue the model loop
- re-emit a synthetic stream back to the client

That keeps the client-side contract streaming-friendly even when the gateway executes part of the tool loop itself.
